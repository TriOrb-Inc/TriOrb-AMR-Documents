    パス: triorb_sensor/sick/triorb_sick_plc_wrapper
    説明: SICK製PLCとのEtherNet/IPデータ交換を行い、速度ベクトルや安全入出力をROS 2メッセージとPLCアセンブリ間で変換するラッパーノードです。

    triorb_sick_plc_wrapper

    ROS 2 node that bridges TriOrb velocity commands to a SICK FlexiSoft PLC. The node translates /drive/std_vector2 values into PLC assembly bytes, mirrors PLC feedback bits, and exposes services for emergency stop, PLC/SLS enabling, and watchdog handling.

    Node interface

    Direction Topic Type Notes
    Subscribe /<prefix>/drive/std_vector2 std_msgs/msg/Float32MultiArray [direction(rad), speed(m/s), rotate(rad/s)]. Direction, speed, and rotate are scaled and packed into the outbound PLC assembly.
    Subscribe /<prefix>/flexisoft/Assem131/in_raw (configurable) std_msgs/msg/UInt8MultiArray PLC inbound assembly. topic_from_plc selects the topic; sub_basic_index chooses the byte that holds basic status bits.
    Publish /<prefix>/flexisoft/Assem141/out_raw (configurable) std_msgs/msg/UInt8MultiArray PLC outbound assembly populated with scaled velocity vectors and control bits. topic_to_plc selects the topic.
    Publish /<prefix>/except_handl/node/add std_msgs/msg/String Registers the node with the global exception handler when coming online.
    Publish /<prefix>/triorb/error/str/add std_msgs/msg/String Emits error diagnostics (e.g., PLC buffer overruns or scaling limits).
    Publish /<prefix>/triorb/warn/str/add std_msgs/msg/String Emits warnings (e.g., short PLC assemblies).
    Service /<prefix>/srv/plc/emergency_stop/request std_srvs/srv/Trigger Raises the PLC emergency-stop bit once and acknowledges the request.
    Service /<prefix>/srv/plc/change_to_plc_off std_srvs/srv/SetBool true → request PLC deactivate, false → cancel the request.
    Service /<prefix>/srv/sls/change_to_sls_off std_srvs/srv/SetBool true → request SLS monitor OFF, false → SLS ON.

    All topics are prefixed with ROS_PREFIX (if set) through the GET_TOPIC_NAME() helper so the node can be namespaced without remapping each interface.

    Repository layout

    • include/sick_plc_wrapper.hpp – Node declaration, PLC bitfield definitions, and helpers.
    • src/sick_plc_wrapper.cpp – Implementation of the bridge, scaling logic, and services.
    • src/ct.cpp, src/sha256.cpp – License verification helpers.
    • launch/plc_wrapper.launch.xml – Example parameterization for FlexiSoft assemblies.

    Key features

    • Bidirectional byte-array interface to SICK FlexiSoft PLC assemblies with configurable offsets per field.
    • Scaling of TriOrb drive vectors into PLC-friendly integer representations (direction 0.1°, speed cm/s, rotate °/s).
    • Watchdog timer that toggles a Jetson → PLC bit while mirroring PLC watchdog responses.
    • Exception-handler publishers for registration, warnings, and errors so operators see PLC issues in a unified feed.
    • Runtime services to request emergency-stop, PLC deactivate, and SLS disable without touching the PLC UI.
    • Startup guard that waits (up to 10 s) for both PLC topics to be connected before proceeding, emitting warnings if the FlexiSoft driver is absent.

    Configuration

    Parameters can be supplied via the launch file or CLI: - topic_to_plc (string, default /flexisoft/Assem141/out_raw) – Outbound PLC assembly topic. - topic_from_plc (string, default /flexisoft/Assem131/in_raw) – Inbound PLC assembly topic. - assembly_size_to_plc (int, default 10) – Total byte length of the outbound array; must cover every configured index. - speed_index, direction_index, rotate_index (int) – Byte offsets (little-endian) where scaled data is written. Bounds are checked at runtime. - pub_basic_index, sub_basic_index (int) – Byte offsets that carry the basic control/status bitfields described below. - scale_drive_vector_speed, scale_drive_vector_direction, scale_drive_vector_rotate (double) – Unit conversions applied before packing (defaults: 100.0, 10.0, 180.0/M_PI). - watchdog_interval_ms (int, default 500) – Period for toggling watchdog_request_from_jetson.

    PLC basic bit layout

    • Publish byte (PubBasicData, defaults to watchdog response off / emergency-stop asserted): bit0 watchdog response → PLC, bit1 watchdog request from Jetson, bit2 emergency stop request, bit3 deactivate request, bit4 SLS OFF request, bit5-7 reserved.
    • Subscribe byte (SubBasicData): bit0 watchdog request from PLC, bit1 watchdog response from PLC, bit2 emergency stop active, bit3 power cut active, bit4 move permitted, bit5-7 reserved.

    Building

    colcon build --packages-select triorb_sick_plc_wrapper
source install/setup.bash

    Running

    ## Launch with the default FlexiSoft wiring
ros2 launch triorb_sick_plc_wrapper plc_wrapper.launch.xml

## Run directly while overriding topics/indices
ros2 run triorb_sick_plc_wrapper sick_plc_wrapper \
  --ros-args \
  -p topic_to_plc:=/flexisoft/Assem141/out_raw \
  -p topic_from_plc:=/flexisoft/Assem131/in_raw \
  -p assembly_size_to_plc:=10

    Ensure /drive/std_vector2 and the FlexiSoft PLC topics are available before starting; the node aborts if Pub/Sub connections are missing for 10 seconds. Provide /install/license.yaml so the built-in KeyCertification check can validate the runtime license.

    Notes / Best practices

    1. Keep assembly_size_to_plc consistent with the FlexiSoft configuration; an undersized buffer prevents the node from writing scaled vectors.
    2. Tune the scale_* factors and indices to match the PLC project any time FlexiSoft addresses change.
    3. Monitor /triorb/error/str/add for out-of-range speeds/angles; these indicate mis-scaled commands or upstream bugs.
    4. When testing, call ros2 service call /srv/plc/emergency_stop/request std_srvs/srv/Trigger {} to validate the PLC reacts as expected.
    5. Namespacing the node? Just set ROS_PREFIX=myrobot before launch to scope every topic/service automatically.
