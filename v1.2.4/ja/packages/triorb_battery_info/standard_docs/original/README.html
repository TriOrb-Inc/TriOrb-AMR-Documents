Package: triorb_battery_info¶

概要¶

CAN バスから取得したバッテリー情報を集約し、ROS 2 のバッテリー状態トピックとして publish するパッケージです。

主な機能¶

  • CAN フレームから SOC・電圧・電流情報をデコード

  • モジュール構成に応じて BatteryStatus を集約

  • 直列 / 並列パックに応じた総電圧計算と publish

補足¶

責務範囲¶

本ノードの責務¶

  • バッテリー SOC / モジュール IV の CAN デコード

  • モジュール数 1 個以上への対応

  • パック構成（直列 / 並列）の考慮

  • パック全体電圧の算出

  • BatteryStatus メッセージの publish

本ノードの非責務¶

  • バッテリー保護・遮断制御

  • 異常判定・フェイルセーフ

  • 充放電制御

  • CAN 通信エラーの復旧処理

  • remaining_percent はそのまま使用

モジュール IV フレーム¶

項目

4 byte 以上

Payload 定義

Byte

内容

0-1

電流 raw

2-3

電圧 raw

変換式

current[A] = (raw - 0x8000) * 0.01119
voltage[V] = raw * 4.8832 / 1000

パラメータ設定¶

最小構成例¶

/**:
  ros__parameters:
    id_soc: 0x053
    module_iv_ids: [0x056, 0x076]
    pack_topology: series

パック電圧算出ルール¶

直列（series）¶

total_voltage = 各モジュール電圧の合計

並列（parallel）¶

total_voltage = 代表電圧

代表電圧は parallel_voltage_method により決定します。

実行方法¶

colcon build --packages-select triorb_battery_info
source install/setup.bash

ros2 run triorb_battery_info battery_status \
  --ros-args --params-file /ws/params/battery_parallel.yaml

動作仕様・注意点¶

  • モジュール数 1 個でも動作可能

  • module_iv_ids が空の場合、ノードは起動失敗

  • DLC 不足の CAN フレームは無視

  • タイムアウトによる無効化処理は未実装

  • pack_topology は全モジュール共通前提

内部設計メモ¶

  • モジュール数は module_iv_ids のサイズから自動決定

  • ID -> モジュール index は起動時に固定マッピング

  • パラメータは起動時のみ反映（動的変更なし）
